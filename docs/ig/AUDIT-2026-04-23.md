## Contexto del bug HIGH (ig-discover:56-67)

El `payloadTemplate` que se envía a Apify incluye:
```json
{
  "eventType": "{{eventType}}",
  "eventData": { "actorRunId": "{{actorRunId}}" },
  "sourceRef": "<hashtag>",
  "signature": "<APIFY_WEBHOOK_SECRET plaintext>"
}
```

El handler en `webhooks/apify/route.ts` hace:
```ts
const signature = req.headers.get('apify-webhook-signature')
// HMAC verification against this header
```

Apify no envía automáticamente el header `apify-webhook-signature`. El campo `signature` en el body body es texto plano y nunca se verifica. Opciones para SESSION-07:
- A) Verificar `body.signature === APIFY_WEBHOOK_SECRET` (más simple, menos seguro)
- B) Configurar Apify con signing key real y ajustar el header esperado
- C) Poner el secret como query param en `requestUrl` y verificarlo allí
